# Critique — Full site (B20 launch hub)

## Anti-Patterns Verdict
Mostly avoids slop. Distinctive: liquid-metal shader, glass orb logo, per-token coin avatars, launchpad ticker. Detector: only "overused-font" (Inter body). Glassmorphism is heavy but it's the committed brand concept. No gradient text, no per-section eyebrow, no identical card grids.

## What's Working
1. Hero now proportional (centered headline) + premium liquid + live "just launched" ticker.
2. Real on-chain data everywhere (explore shows real Sepolia launches; honest mainnet gating).
3. Mobile is clean (nav, coin preview, wizard stack well).

## Priority Issues
- [P1] Report page (/token, /sepolia/token) renders in ~31.6s — effectively "broken" to users. Cause: on-chain role-event scan + B20Created creation-block scan from block 0 over public RPC (getLogs range limits → retries/timeouts). The Verify pillar's depth page. Fix: bound/lazy-load the role scan, add per-call timeouts, cache, or drop role history to on-demand. → harden / optimize.
- [P1] Brand name "B20 Proof" no longer fits — it's a launch hub now (launch + explore + verify), not a proof/scanner. Undersells the primary action. → rebrand.
- [P2] Body font is Inter (overused AI-default per detector + brand reflex-reject). Weakest part of an otherwise distinctive type system (condensed display + mono). → typeset.
- [P2] Header cramps at ~1000-1100px: nav + "Built on Base" + Connect Wallet collide before the mobile stack kicks in. Needs an intermediate breakpoint. → adapt.
- [P3] Two hero compositions diverge (launch-led home vs verify hero); report/verify use older glass styling vs the launchpad coins. Cohesive but not identical.

## Persona Red Flags
- Trader (skeptical): clicks a token in Explore -> report page hangs 30s -> assumes scam/broken, leaves. The slow report directly undermines trust on the verify path.
- First-timer: name "Proof" suggests a checker, not a launchpad; the primary action (launch) is hidden behind a verification-sounding brand.

## Questions to Consider
- Should the report page show identity + factory verdict instantly and lazy-load role history separately?
- Does the brand lead with "launch" in the name, or stay neutral across launch/verify?
