## Anti-Patterns Verdict
Mixed. Distinctive liquid-metal hero (not slop) but absolute-ban tells in CSS: gradient-text logo (.brand-b20), side-stripe accents (.feature-strip h3::before + proof-label ticks), glassmorphism-as-default, two identical 3-up card grids (proof + feature-strip). detect.mjs on markup = 0 findings (clean JSX), but it does not scan CSS where the tells live. Contrast audited in-browser and PASSES (6.42 / 8.68 / 5.88 / 10.8).

## What's Working
1. Liquid-metal WebGL hero — committed, distinctive, on-brand; top-to-bottom continuity.
2. Verdict-first hierarchy — lime "FACTORY CONFIRMED" is the right focal point.
3. Honest failure states (Status "Live on Base" / "Verification Unavailable").

## Priority Issues
- [P1] Hero result is a static mockup that reads as a real verdict; contradicts PRODUCT principle #2 ("Read, don't claim"). Fix: mark as example or run a real sample. → clarify/shape.
- [P1] Two identical 3-up card grids + glass-as-default flatten hierarchy. Fix: glass for data only, different form for the explanatory feature-strip. → layout/quieter.
- [P2] Absolute-ban CSS tells: gradient-text logo, side-stripe accents. Fix: solid logo, full borders/tints/leading icon. → polish.
- [P2] Empty recent panels dominate the lower-right with "Recent checks will appear here". Fix: seed sample addresses or collapse until populated. → onboard.
- [P3] Dead code: hero-shader.tsx, hero-stage.tsx, legacy .hero/.hero-panel/.checker CSS. → cleanup.
